# Review Checklist Template

## General programming

- [ ] Existing implementations were searched.
- [ ] No unnecessary rewrite.
- [ ] No duplicate helper/parallel implementation.
- [ ] No nontrivial magic numbers outside config/constants authority.
- [ ] No secrets or local paths.
- [ ] Generated outputs were not hand-edited.
- [ ] Tests assert meaningful behaviour.
- [ ] Public/cross-layer APIs remain compatible or migration is documented.
- [ ] Suppressions are narrow and reasoned.
- [ ] Technical debt delta is recorded if applicable.

## Project-specific overlay

- [ ] Project authority map respected.
- [ ] Project verification command run.
- [ ] Project-specific generated-file rules followed.
- [ ] Project-specific language/framework overlays followed.
